# Lint Fix

프로젝트의 ESLint 경고와 에러를 모두 찾아 수정한다.

## 절차

### 1단계: 환경 파악

- 패키지 매니저 감지 (pnpm/npm/yarn/bun — lock 파일 기준)
- eslint 설정 파일 존재 여부 확인 (eslint.config.*, .eslintrc.*)
- eslint가 없으면 "ESLint 설정이 없습니다" 안내 후 종료

### 2단계: auto-fix 실행

```bash
{패키지매니저} eslint --fix . 2>&1
```

auto-fixable한 경고/에러를 자동 수정한다.

### 3단계: 남은 이슈 파악

```bash
{패키지매니저} eslint . 2>&1
```

auto-fix 후에도 남은 경고/에러 목록을 수집한다.

남은 이슈가 없으면 결과 리포트 출력 후 종료.

### 4단계: 수동 수정

남은 이슈를 **파일별로 그룹화**하여 하나씩 수정한다:

1. 해당 파일을 읽는다
2. 에러/경고의 원인을 파악한다
3. 코드를 수정한다
4. 다음 파일로 넘어간다

**주의:**
- lint 규칙을 disable하는 주석(`eslint-disable`)은 사용하지 않는다 — 코드를 고쳐서 해결한다
- 규칙 자체가 프로젝트에 맞지 않는 경우에만 eslint 설정 변경을 제안한다
- 수정으로 인해 기존 동작이 바뀌지 않도록 주의한다

### 5단계: 검증

모든 수정 후 eslint를 다시 실행하여 경고/에러가 0인지 확인한다.

```bash
{패키지매니저} eslint . 2>&1
```

### 6단계: 결과 리포트

```
LINT FIX REPORT
===============
auto-fix:   {N}개 수정
수동 수정:   {N}개 수정
남은 이슈:   {N}개 (있으면 사유 설명)
변경 파일:   {파일 목록}
```
